                             Функция url_for и переменные URL-адреса

1. Функция url_for позволяет генерировать url-адрес по имени функции-обработчика.
При импорте добавляем url_for и вызываем ее в обработчике главной страницы и обработчике
о сайте. Соответственно, url_for будет возвращать адрес нашей страницы.
!!! Стоит учесть, что эта функция будет возвращать адрес, записанный в последнем декораторе.
2. Вызов этой функции непосредственно связан с контекстом запроса и вне этого контекста
она попросту не будет работать.
3. Фо Flask можно протестировать менеджер запросов не запуская локальный сервер.
    with app.test_request_context():
        print(url_for('index'))
4. Функция url_for обращается к current app, затем этот запрос ищет, с каким приложением
он ассоциирован и возвращает его адрес.
5. URL-адреса можно делать переменными(динамическими)
Используется следующий синтаксис: @app.route("/url/<path:variable>")
Конвертер path говорит нам о том, что все, что будет написано после username, нужно
будет поместить.
Существуют еще способы описания:
int - должны прсутствовать только цифры
float - можно записывать число с плавающей точкой
path - Можно использовать любые допустимые символы URL плюс символ слеша '/'
